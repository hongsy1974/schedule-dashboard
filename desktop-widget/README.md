# WorkFlow 바탕화면 위젯

WorkFlow Portal의 "해야할 일" 목록을 윈도우 바탕화면에 항상 띄워두는 작은 프로그램이에요.
사이트와 같은 데이터(Firestore)를 그대로 보여주기 때문에, 사이트에서 업무를 등록/수정하면
위젯에도 실시간으로 반영돼요.

## 1단계. Node.js 설치하기 (한 번만 하면 됨)

이 프로그램을 실행하려면 컴퓨터에 "Node.js"라는 프로그램이 필요해요. 이미 설치되어 있다면
이 단계는 건너뛰어도 돼요.

1. https://nodejs.org 접속
2. 화면에 크게 보이는 초록색 버튼(LTS 버전) 클릭해서 설치 파일 다운로드
3. 다운로드된 파일을 실행하고, 계속 "다음(Next)"만 눌러서 설치 완료

## 2단계. 이 폴더 다운로드하기

1. https://github.com/hongsy1974/schedule-dashboard 접속
2. 초록색 **Code** 버튼 클릭 → **Download ZIP** 클릭 (저장소 전체가 zip 파일로 다운로드돼요)
3. 다운로드된 zip 파일을 컴퓨터의 원하는 위치(예: 바탕화면이나 문서 폴더)에 압축 풀기
4. 압축 푼 폴더 안에서 `desktop-widget` 폴더를 찾습니다 (이 프로그램에 필요한 건 이
   폴더뿐이에요 — 나머지는 사이트 소스코드라 필요 없어요)

## 3단계. 프로그램 실행하기

1. 방금 압축을 푼 `desktop-widget` 폴더를 엽니다.
2. 폴더 안 빈 곳에서 **Shift 키를 누른 채 마우스 오른쪽 버튼 클릭** → "여기에 PowerShell 창 열기"
   (또는 "여기서 명령 프롬프트 열기") 선택
3. 열린 검은/파란 창에 아래 글자를 입력하고 Enter (최초 1회만, 몇 분 걸릴 수 있어요)
   ```
   npm install
   ```
4. 설치가 끝나면 아래 글자를 입력하고 Enter
   ```
   npm start
   ```
5. 화면 오른쪽 위에 어두운 카드 모양의 위젯이 나타나요 🎉

다음에 다시 켜고 싶을 땐 3번(`npm install`)은 건너뛰고 4번(`npm start`)만 하면 돼요.

## 위젯 사용법

- **위치 옮기기**: 위젯 위쪽(제목이 있는 부분)을 마우스로 눌러서 드래그하면 원하는 곳으로
  옮길 수 있어요. 다음에 켤 때도 옮긴 위치를 기억해요.
- **보기 전환**: 오른쪽 위 📋(리스트) / 📆(주간) / 📅(달력) 버튼으로 전환할 수 있어요.
  마지막으로 본 화면을 기억해서 다음에 켤 때도 그 화면으로 열려요.
  - 📋 리스트: "해야할 일"과 같은 방식(긴급도×2 + 중요도 + 마감가중)으로 정렬되고,
    완료 처리된 업무와 개인 일정은 표시되지 않아요.
  - 📆 주간: 사이트의 주간 일정과 똑같은 형식이에요. 하루 이상 걸리는 긴 업무(10일
    이상)는 요일 위에 막대로 이어서 표시되고, 나머지는 날짜 칸마다 업무 이름이 보여요.
    색은 상태에 따라 달라요(초록=완료, 빨강=지연, 주황=그 외). 개인 일정도 함께
    표시돼요. ‹ › 로 지난 주/다음 주 이동, ● 로 이번 주 복귀.
  - 📅 달력: 사이트의 월간 일정처럼 날짜마다 업무 이름이 보여요(안 들어가면 "+N"으로
    표시). 색은 상태에 따라 달라요(초록=완료, 빨강=지연, 주황=그 외). 개인 일정도
    여기서는 함께 표시돼요. ‹ › 로 다른 달 이동, ● 로 이번 달 복귀. 업무명이 보여야
    해서 리스트 보기보다 창이 훨씬 넓어요 — 전환할 때마다 창 크기가 자동으로 바뀌어요.
- **📌 항상 위에 표시**: 기본값은 꺼져 있어요(다른 작업을 방해하지 않도록). 눌러서 켜면
  다른 창들보다 항상 위에 떠 있어요. 설정은 기억돼서 다음에 켤 때도 유지돼요.
- **▁ 접기 / ▢ 펼치기**: 누르면 위젯이 제목 줄만 남기고 작아져요. 다시 누르면 원래
  크기로 돌아와요. 이 상태도 다음에 켤 때 기억돼요.
- **닫기**: 오른쪽 위 × 버튼
- **메뉴**: 오른쪽 위 ⋯ 버튼 → 사이트 열기 / 항상 위에 표시 켜기·끄기 / 위젯 종료
- **업무·날짜 클릭 / "WorkFlow Portal 열기" 버튼**: 기본 브라우저로 실제 사이트가 열려요.

## 컴퓨터를 켤 때마다 자동으로 뜨게 하고 싶다면 (선택 사항)

1. `Win + R` → `shell:startup` 입력 후 Enter (윈도우 시작프로그램 폴더가 열려요)
2. 아래 내용으로 `workflow-widget.bat` 라는 이름의 파일을 그 폴더 안에 만들어주세요.
   (메모장에서 아래 내용을 붙여넣고, 파일 저장할 때 파일 이름을 `workflow-widget.bat`로,
   "파일 형식"은 "모든 파일"로 지정하면 돼요.)
   ```
   cd /d "여기에 desktop-widget 폴더의 실제 경로를 붙여넣으세요"
   npm start
   ```
3. 컴퓨터를 다시 시작하면 자동으로 위젯이 켜져요.

## 문제가 있을 때

- **"연결 실패"라고 나와요**: 인터넷 연결을 확인해 주세요. 회사 네트워크에서 구글 서비스
  (gstatic.com, firestore 등)를 막아두었다면 뜨지 않을 수 있어요.
- **`npm install`이 오류를 내요**: Node.js가 제대로 설치됐는지 확인해 주세요 (PowerToys/명령
  창을 새로 열어서 다시 시도해보는 것도 도움이 돼요).
- 그 외 문제는 화면을 캡처해서 알려주시면 같이 확인해볼게요.
